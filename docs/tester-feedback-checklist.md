# Tester Feedback Checklist

Use this checklist when sending the v0.1 demo to first testers.

Live demo:

- https://english-roleplay-party.vercel.app/onboarding

## Quick tester script

Send this message:

> 这是一个 3 分钟英语小剧场 demo。你不用准备，选一个角色，点麦克风说一句英文，系统会给你转写、评分和表达卡。玩完告诉我哪里好玩、哪里卡住。

## Core questions

Ask testers:

1. Did you understand what this app does within 10 seconds?
2. Did you know what to tap first?
3. Did the app feel like a game rather than a class or exam?
4. Did the voice flow feel natural?
5. Did the rescue line make you less nervous?
6. Did the scoring feel encouraging?
7. Did the result page make you want to play again?
8. Would you share this with a friend who is learning English?
9. What felt confusing or too slow?
10. What would make you come back tomorrow?

## Watch for behavior

During a test session, observe:

- whether the tester reads the first screen
- whether they hesitate before pressing the mic
- whether they understand that they can stop recording
- whether they notice the rescue line
- whether they read the AI feedback
- whether they smile or react to the result title
- whether they ask what to do next

## Blocking issues

Create a GitHub issue if any tester hits:

- page crash
- dead button
- microphone permission loop
- no visible result after recording
- unreadable mobile layout
- confusing route or missing navigation

## Nice-to-have feedback

Collect quotes for product copy:

- “I liked...”
- “I was confused by...”
- “I wanted...”
- “I would share it if...”

## Decision after 10 testers

If most testers complete one run:

- continue launch polish
- add a second scenario
- consider WeChat Mini Program exploration

If most testers do not complete one run:

- simplify onboarding
- make mic instructions clearer
- reduce steps before first recording

# Designer outreach messages

Use these templates to contact designers after reviewing their portfolio.

## Short first message

```text
Hi, I am building a mobile-first English speaking roleplay party app called 英语角色派对.

It is a voice-first social/game-like English practice product: users enter short scenes, read or improvise English lines, and receive playful ratings and expression cards.

I am looking for a professional mobile UI/product designer to create a polished Figma design package for the v0.1 core flow. The target style is youthful, bright, social, and game-like — not classroom, SaaS, or random gradient template.

Could you share relevant mobile app / social app / game UI portfolio examples, your estimated timeline, and what deliverables you would include?
```

## Detailed first message

```text
Hi, I am looking for a professional designer for a mobile-first product called 英语角色派对.

Product concept:
Users practice spoken English by acting in short roleplay scenes. Each round shows a clear English line to read or improvise. The app records voice, gives playful feedback, and rewards users with ratings and expression cards.

What I need:
A Figma design package for the core v0.1 mobile experience, including:

1. Onboarding / landing
2. Lobby / story hall
3. Room / team ready page
4. Play / voice acting page
5. Result / rating card
6. Expression cards
7. Share handoff page

Design direction:
- polished consumer mobile app
- youthful and bright
- social party game feeling
- character-forward
- voice-first
- low-pressure for shy English speakers

What to avoid:
- classroom / exam feeling
- SaaS dashboard style
- random gradients
- emoji-heavy placeholder UI
- childish kindergarten cartoon style

Expected deliverables:
- Figma source file
- high-fidelity core screens
- reusable components and states
- color/type/spacing tokens
- exported assets
- engineering handoff notes

Please send:
- relevant portfolio links
- your estimated timeline
- suggested scope and price range
- whether source files and commercial usage rights are included
```

## Personal network referral message

```text
I am looking for a strong mobile product/UI designer for my app 英语角色派对.

It is a voice-first English roleplay party game: users act out short scenes in English, get ratings, and collect expression cards.

I need someone with good consumer mobile app taste, ideally with social/game/gamified product experience. The deliverable is a professional Figma design package for the core mobile screens, not just decoration.

Do you know anyone with strong mobile app or game/social UI work?
```

## Follow-up after no reply

```text
Hi, just following up on my previous message about the 英语角色派对 mobile UI design project.

I am currently shortlisting designers. If this type of mobile consumer app / game-like UI project is relevant to you, please send a portfolio link and rough availability. If not, no worries.
```

## Follow-up after portfolio received

```text
Thanks for sending your portfolio.

The most relevant parts for this project are mobile consumer app polish, social/game-like interaction, character or avatar systems, and clear engineering handoff.

Could you also share:

1. one mobile app project where you designed the full flow
2. whether you usually provide component states and design tokens
3. your preferred revision process
4. whether commercial usage rights and Figma source files are included
```

## Request for proposal

```text
I would like to understand how you would scope the first design phase.

Please propose:

1. screens you would include
2. visual direction/moodboard step
3. number of revisions
4. estimated timeline
5. final deliverables
6. price or price range
7. what you need from me before starting

The first phase should focus on the v0.1 core loop rather than future multiplayer or Mini Program work.
```

## Pause / rejection message

```text
Thanks for taking the time to share your work and discuss the project.

For this phase, I am prioritizing designers with very close mobile consumer app / social game UI experience and strong engineering handoff examples, so I will not move forward right now.

I appreciate your time and may reconnect if the scope changes later.
```

## Acceptance / next-step message

```text
Thanks. I would like to move forward to the next step.

Before starting, let's confirm:

1. scope and screen list
2. timeline and milestones
3. revision rounds
4. Figma source ownership
5. asset ownership / commercial usage rights
6. final engineering handoff deliverables
7. payment schedule

Once confirmed, I will send the product design brief, handoff checklist, current MVP link, and reference direction.
```

## What not to say

Avoid saying:

- "Just make it look better"
- "Copy this exact reference"
- "Use anything you want"
- "Assets do not matter"
- "We can fix it in code later"

The designer needs a clear product brief, not vague taste feedback.

## Interview questions

Ask these before selecting a designer.

### Product understanding

1. How would you make this feel like a social party game instead of an education app?
2. How would you reduce fear for users who are shy about speaking English?
3. Which screen do you think matters most for first-time activation?
4. How would you make the result page feel shareable?

### Mobile product design

1. Can you show a mobile app flow you designed end to end?
2. How do you handle small-screen density and hierarchy?
3. How do you design bottom navigation and recurring components?
4. What viewport size do you usually design for?

### Game/social UI

1. Have you designed game-like rewards, levels, badges, or cards?
2. How would you avoid making the app look childish?
3. How would you use characters or avatars without relying on random emoji?
4. What visual elements would make the lobby feel alive?

### Voice-first interaction

1. Have you designed audio, recording, or voice input flows before?
2. How would you communicate recording, processing, and fallback states?
3. How would you make the user know exactly what to say each turn?
4. How would you handle microphone permission denial visually?

### Design system and handoff

1. Do you provide Figma components with states?
2. Do you define color, typography, spacing, and radius tokens?
3. How do you prepare assets for engineers?
4. Can you include notes for edge cases and responsive behavior?
5. Can you show an example of an engineering handoff file?

### Commercial and ownership

1. Are Figma source files included?
2. Are commercial usage rights included?
3. Will any third-party assets be used?
4. Will any AI-generated assets be used?
5. Can assets be edited later by another designer?

## Selection checklist

Before selecting a designer, confirm:

- portfolio matches mobile consumer app quality
- designer understands product direction
- designer accepts the core screen scope
- timeline is realistic
- revision rounds are clear
- Figma source is included
- commercial usage rights are clear
- asset sources are clear
- final handoff includes engineering notes
- pricing and payment schedule are clear

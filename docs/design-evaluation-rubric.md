# Design evaluation rubric

Use this rubric to evaluate designer candidates, style proposals, and final design delivery.

## Scoring method

Score each item from 1 to 5.

- 1 = unacceptable
- 2 = weak
- 3 = acceptable
- 4 = strong
- 5 = excellent

A proposal should not move to engineering unless it passes the minimum thresholds below.

## Candidate portfolio rubric

| Category | 1 | 3 | 5 |
| --- | --- | --- | --- |
| Mobile app polish | Looks like rough templates | Looks usable | Looks like a shipped consumer app |
| Social/game feeling | Feels static or SaaS-like | Has some playful moments | Feels energetic and social without clutter |
| Character direction | No character/illustration skill | Can use avatars/cards | Strong character-forward product taste |
| Voice-first clarity | Does not understand audio flow | Can design basic mic states | Makes voice action obvious and safe |
| Youthful taste | Childish or messy | Moderately bright | Polished, young, modern |
| Design system | No components | Some components | Clear reusable system with states |
| Engineering handoff | Static images only | Basic Figma | Tokens, states, assets, notes |
| Product judgment | Just decorates | Understands flow | Improves activation and confidence |

Minimum candidate score:

- 30 out of 40 total
- must score at least 4 in mobile app polish
- must score at least 4 in engineering handoff
- must score at least 3 in voice-first clarity

## Style direction rubric

Evaluate the first visual direction before approving full screen production.

| Category | What excellent looks like | Score |
| --- | --- | --- |
| First impression | Looks like a real mobile consumer app | 1-5 |
| Product fit | Clearly supports English roleplay party concept | 1-5 |
| Energy control | Bright and fun without becoming cheap | 1-5 |
| Character presence | Characters/avatars feel intentional | 1-5 |
| Layout discipline | Screen density fits mobile usage | 1-5 |
| Typography | Hierarchy is clear and readable | 1-5 |
| Color system | Palette is distinct but controlled | 1-5 |
| Avoids bad patterns | No emoji spam, random gradients, or SaaS look | 1-5 |

Minimum style direction score:

- 32 out of 40 total
- no score below 3

## Core screen rubric

### Onboarding / landing

Pass criteria:

- user understands the product in 10 seconds
- user sees the product as fun, not scary
- CTA is clear
- role/avatar setup feels lightweight
- copy reassures shy speakers

Score:

| Item | Score |
| --- | --- |
| 10-second comprehension | 1-5 |
| CTA clarity | 1-5 |
| Emotional reassurance | 1-5 |
| Character-forward feeling | 1-5 |
| Mobile polish | 1-5 |

### Lobby / story hall

Pass criteria:

- feels like a game/social app lobby
- featured scenario is attractive
- scenario list is easy to scan
- future locked sections do not clutter
- navigation is clear

Score:

| Item | Score |
| --- | --- |
| Game lobby feeling | 1-5 |
| Featured scenario quality | 1-5 |
| Scenario card clarity | 1-5 |
| Navigation clarity | 1-5 |
| Visual consistency | 1-5 |

### Play / voice acting page

Pass criteria:

- the English line is immediately visible
- mic action is obvious
- recording states are clear
- fallback feels helpful
- screen keeps user focused on one turn

Score:

| Item | Score |
| --- | --- |
| Voice-first clarity | 1-5 |
| Script readability | 1-5 |
| Recording state design | 1-5 |
| Feedback clarity | 1-5 |
| Low-pressure feeling | 1-5 |

### Result / rating card

Pass criteria:

- result feels rewarding
- rating feels like game achievement, not exam score
- share/screenshot appeal is strong
- expression cards feel collectible
- next action is clear

Score:

| Item | Score |
| --- | --- |
| Reward feeling | 1-5 |
| Rating design | 1-5 |
| Shareability | 1-5 |
| Expression card appeal | 1-5 |
| Replay CTA clarity | 1-5 |

## Engineering handoff rubric

| Category | Required for 5/5 | Score |
| --- | --- | --- |
| Figma organization | Pages, frames, and components are named clearly | 1-5 |
| Tokens | Colors, type, spacing, radii, shadows documented | 1-5 |
| Component states | Default, pressed, disabled, loading, error states included | 1-5 |
| Asset export | Icons/illustrations exportable and named | 1-5 |
| Responsive rules | Mobile viewport and safe-area behavior clear | 1-5 |
| Implementation notes | Edge cases and behavior notes included | 1-5 |

Minimum engineering handoff score:

- 24 out of 30 total
- no score below 3

## Rejection conditions

Reject or request revision if any of these happen:

- design relies on emoji as main artwork
- main visual identity is random gradients
- first screen does not explain the product
- play screen hides or weakens the English line
- mic state is unclear
- result page feels like a school report card
- no component states are provided
- no editable source file is provided
- assets have unclear ownership rights

## Approval levels

### Approved for exploration

- candidate or direction is promising
- not ready for engineering
- needs refinement

### Approved for design production

- visual direction is strong
- product fit is clear
- designer can proceed to core screens

### Approved for engineering

- final Figma is complete
- handoff checklist is complete
- assets are ready
- implementation can start screen by screen

## Final product bar

The design should make the product feel like:

> A polished mobile social game for practicing spoken English through roleplay.

Not:

> A random template, classroom app, SaaS dashboard, or emoji-filled prototype.

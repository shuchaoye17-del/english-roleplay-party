# Designer sourcing plan

This document turns the professional design brief into an execution plan for finding and working with a designer or design team.

## Objective

Find a designer who can create a polished mobile-first consumer app design for English Roleplay Party.

The designer should not only make screens look nicer. They must understand:

- mobile social products
- game-like interfaces
- character-forward visual systems
- voice-first user flow
- low-pressure learning experiences
- engineering handoff discipline

## Recommended designer profile

Prioritize designers with portfolio evidence in at least two of these areas:

- consumer mobile apps
- social apps
- mobile games or gamified apps
- youth-oriented entertainment products
- character/avatar-based products
- education products that do not look like classroom tools
- onboarding and activation flows
- design systems for engineering handoff

Nice to have:

- illustration direction experience
- avatar / character art direction
- Figma component library experience
- experience designing for Chinese mobile users
- experience with WeChat Mini Program constraints

## Where to source designers

Use multiple channels rather than relying on one.

### Personal network

Best for speed and trust.

Ask for:

- mobile app product designers
- game UI designers
- social product designers
- designers who can work in Figma and hand off to engineers

### Portfolio platforms

Search for designers with strong mobile consumer work.

Look for:

- complete app case studies
- actual product flows, not only landing pages
- mobile screen systems
- game/social/creator tools
- clear before/after design thinking

### Freelance marketplaces

Useful if scope is tightly defined.

Only proceed if the designer can show:

- relevant mobile UI portfolio
- clear delivery timeline
- source file handoff
- revision process
- commercial usage rights

### Design studios

Useful if budget allows and brand polish matters.

Best for:

- brand identity
- app UI system
- illustration direction
- launch marketing visuals

## Portfolio evaluation criteria

Score each candidate from 1 to 5.

| Area | What to look for | Score |
| --- | --- | --- |
| Mobile polish | Screens look like real shipped mobile apps | 1-5 |
| Visual hierarchy | User can understand the screen quickly | 1-5 |
| Consumer energy | Work feels lively, not enterprise/SaaS | 1-5 |
| Game/social feel | Understands playful interaction and reward | 1-5 |
| Character direction | Can handle avatar/illustration-forward products | 1-5 |
| Design system | Uses components, states, tokens | 1-5 |
| Handoff quality | Can prepare specs engineers can use | 1-5 |
| Taste fit | Avoids cheap gradients, emoji spam, clutter | 1-5 |

Strong candidate threshold:

- total score 30+ out of 40
- no score below 3 in mobile polish, hierarchy, taste fit, or handoff quality

## Red flags

Avoid designers who:

- only show static posters or landing pages
- cannot show mobile product flows
- rely heavily on emoji and random gradients
- cannot explain component states
- cannot hand off Figma source files
- do not understand engineering constraints
- push a cute childlike style without product fit
- make everything look like a SaaS dashboard
- cannot commit to revision rounds
- are vague about commercial usage rights

## First outreach message

```text
Hi, I am building a mobile-first English speaking roleplay party app called 英语角色派对.

The product is a voice-first social/game-like English practice experience: users enter short scenes, read or improvise English lines, and receive playful rating and expression cards.

I am looking for a professional mobile UI/product designer to create a Figma design package for the core v0.1 screens. The direction should feel youthful, polished, social, and game-like — not like a classroom app, SaaS dashboard, or random gradient template.

Core screens include onboarding, lobby/story hall, room/team ready page, play/voice acting page, result/rating card, expression cards, and share handoff page.

I can provide a product brief, current MVP link, and engineering handoff checklist. Please share relevant mobile app or game/social UI portfolio examples, estimated timeline, and what deliverables you would include.
```

## What to send candidates

Send these repository docs:

- `docs/professional-design-brief.md`
- `docs/design-handoff-checklist.md`
- current preview link if accessible
- screenshots of current MVP
- the visual reference image approved by product direction

Also explain:

- current UI is not final
- visual direction must be professionally redesigned
- engineering will implement after Figma is approved

## Design scope for first contract

Recommended first scope:

### Required high-fidelity screens

- onboarding / landing
- lobby / story hall
- play / voice acting page
- result / rating card

### Required medium-fidelity screens

- room / team ready page
- expression cards
- share handoff page

### Required system pieces

- colors
- typography
- buttons
- cards
- badges
- bottom navigation
- microphone states
- scenario card
- rating card

Why this scope:

- it covers the core user loop
- it avoids overpaying for later features before validation
- it gives engineering enough to implement a coherent upgrade

## Suggested milestones

### Milestone 1: style direction

Deliver:

- moodboard
- visual references
- 1-2 rough style directions
- proposed color/type direction

Decision:

- choose one direction before screen production

### Milestone 2: core flow design

Deliver high-fidelity:

- onboarding
- lobby
- play
- result

Decision:

- approve core flow before expanding to secondary screens

### Milestone 3: component and state design

Deliver:

- buttons
- cards
- badges
- nav
- mic states
- result/rating states
- expression card states

Decision:

- engineering confirms implementability

### Milestone 4: final handoff

Deliver:

- Figma source
- exported assets
- design tokens
- screen notes
- component states
- usage rights confirmation

Decision:

- ready for engineering implementation PRs

## Budgeting guidance

Do not choose purely by lowest price. A cheap design that forces rework is expensive.

Evaluate by:

- portfolio match
- product taste
- ability to design systems
- communication clarity
- handoff discipline
- timeline reliability

## Ownership and usage rights

Before work starts, confirm in writing:

- commercial usage rights
- source file ownership or license
- asset ownership or license
- whether AI-generated assets are used
- whether third-party assets require attribution
- whether illustrations/icons can be edited later

Do not accept final files only as PNG screenshots.

## Final recommendation

Start with 3-5 candidate conversations. Pick the designer only after seeing relevant mobile app work and confirming they can deliver Figma source, component states, and engineering handoff.

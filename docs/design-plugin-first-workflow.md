# Design plugin first workflow

## Decision

Major visual design work must use professional design tools before engineering implementation.

The project will not use code-only visual invention as the final UI process.

## Responsibilities

### Product / engineering coordination

Responsible for:

- product requirements
- user flow
- interaction logic
- technical feasibility
- implementation
- testing
- deployment

### Design tools

Responsible for:

- visual direction
- layout exploration
- style systems
- component appearance
- visual consistency
- design artifacts

## Current design workflow

1. Define product requirement.
2. Create design brief.
3. Use professional design tooling to explore visual directions.
4. Review selected direction.
5. Create engineering handoff.
6. Implement only approved designs.
7. Validate against design source.

## Current internal design tool

Canva is used as an internal design exploration tool for:

- visual direction boards
- mobile app concept exploration
- shareable design references
- early UI mood exploration

Canva output is a design source/reference, not automatically production UI.

## External designers

External designers are optional.

They may be used when:

- higher visual polish is required
- illustration systems are needed
- brand identity needs professional refinement
- Figma-level production handoff is required

They are not a requirement for every iteration.

## Engineering rule

Do not merge large UI changes without:

- approved design source
- screen reference
- component rules
- asset source
- acceptance criteria

## Avoid

- random CSS redesigns
- emoji as final artwork
- uncontrolled gradients
- changing UI because it "looks fun" without a system
- implementing screenshots without understanding interaction states

## Goal

Move from prototype styling to a professional product design workflow.

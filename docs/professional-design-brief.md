# Professional design brief

## Project

English Roleplay Party / 英语角色派对

## One-liner

用英语演故事，边玩边练口语。

## Product concept

A mobile-first, voice-first English roleplay party game. Users enter short social scenes, choose or receive a character, read or improvise English lines, and receive playful feedback, rating, and expression cards.

The product should feel like a consumer social game, not a classroom tool and not a SaaS dashboard.

## Target users

Primary users:

- young English learners who want to speak but feel shy
- people who like social mobile games
- users who want a low-pressure way to practice spoken English
- friends who may challenge each other through shared short scenes

User emotional state:

- curious but easily embarrassed
- afraid of speaking wrong English
- wants a safe script to read
- wants quick reward and social proof

## Desired feeling

The app should feel:

- energetic
- young
- bright
- playful
- polished
- social
- low-pressure
- party-like
- voice-first

The experience should make the user think:

> This is not an English class. This is a fun little performance game where I can safely speak one line.

## Visual reference direction

The target direction is a polished mobile party app with:

- character-forward UI
- strong story-room cards
- clear stage / performance moments
- expressive result cards
- screenshot-worthy rating screen
- high-quality illustration or avatar assets
- controlled bright color palette
- sticker-like accents used sparingly and intentionally

## What to avoid

Avoid:

- cheap random gradients
- emoji as primary visual identity
- cluttered sticker spam
- SaaS dashboard style
- serious education app style
- classroom / exam feeling
- dark dating-app feeling
- childish kindergarten cartoon style
- unreadable typography
- oversized spacing that wastes mobile screen space
- overly realistic romance imagery

## Core product screens to design

The professional design package should include high-fidelity mobile designs for these screens.

### 1. Onboarding / landing

Goal:

- explain the product in 10 seconds
- make the user want to start a 3-minute demo
- reduce fear of speaking English

Must include:

- product title: 英语角色派对
- one-liner: 用英语演故事，边玩边练口语
- clear CTA: 开一局英语小剧场
- reassurance: 不会说也有台词提示
- role / avatar selection

### 2. Lobby / story hall

Goal:

- feel like a game lobby or story hall
- make the first scenario attractive
- show future expansion without clutter

Must include:

- featured scenario
- scenario list
- tags such as 练口语 / 组队开黑 / 实时互动
- user status area
- bottom navigation

### 3. Room / team ready page

Goal:

- feel like a team room before a short performance game
- make roles and readiness clear

Must include:

- scenario title
- player slots
- role lineup
- ready status
- start button
- AI stand-in concept

### 4. Play / voice acting page

Goal:

- make the user focus on one English line
- make recording feel safe and obvious
- preserve voice-first flow

Must include:

- current role
- current prompt
- visible English line to read
- microphone button states
- recording / processing / fallback states
- AI response or feedback area

### 5. Result / rating card

Goal:

- give emotional reward
- make the user want to screenshot or share
- make the score feel like a game rating, not an exam grade

Must include:

- S/A/B style rating
- title / achievement
- score dimensions
- high-light line
- expression cards earned
- CTA to replay or share

### 6. Expression cards

Goal:

- make English phrases feel collectible
- show practical phrases in a fun way

Must include:

- card title
- phrase
- Chinese meaning
- usage context
- rarity / category style if needed

### 7. Share handoff page

Goal:

- let testers understand the product before entering the app
- provide copy that can be shared to friends

Must include:

- one-liner
- what to test
- what feedback to give
- CTA into demo

## Required asset direction

The design should define:

- avatar style
- character illustration style
- scenario cover style
- badge / sticker style
- rating medal style
- microphone / voice icon style
- expression card style

Assets should feel consistent. Do not mix random emoji, random stock illustrations, and unrelated icon styles.

## Brand basics to define

The design package must define:

- logo / wordmark direction
- primary colors
- secondary colors
- background colors
- typography scale
- icon style
- border radius rules
- shadow depth rules
- card layout system
- bottom navigation style
- button hierarchy

## Voice-first product rules

The design must support these product principles:

- every turn should show the English line directly
- microphone action should be obvious
- user should never feel punished for bad English
- fallback / AI stand-in should feel helpful, not like failure
- result should feel like performance feedback, not school grading

## Design acceptance criteria

A design pass is acceptable only if:

- the first screen communicates the product in under 10 seconds
- the lobby feels like a polished mobile game/social app
- the play screen clearly prioritizes voice input
- the result screen is screenshot-worthy
- visual style is consistent across all screens
- design avoids emoji-heavy placeholder aesthetics
- engineering can implement the system from clear specs

## Engineering implementation note

Do not implement major UI changes from verbal direction alone. Every large UI PR should link to:

- Figma file or design source
- relevant screen name
- component states
- export assets
- acceptance notes
